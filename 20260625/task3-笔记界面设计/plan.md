# Task 3: KnowledgeMap 笔记与神经元关系视图

## 本 task 要完成的内容

在 `/notes` 路由下实现一个完全动态的、力导向交互式知识拓扑图谱系统，用于管理与可视化个人学习笔记。由于是后置编写的开发记录，文档重点突出复杂物理引擎算法的设计与优化。

---

### 【核心重头戏】自研 Verlet 力导向物理引擎与几何避障（极高工作量 / 高代码量 / 极重要）

这是整个任务的核心攻坚点，代码包含大量动态数学计算与物理阻尼模拟：

1. **基本引力与斥力**：
   - 粒子距离平方级磁性斥力（Repulsion）：防止节点过于拥挤。
   - 重力拉拽（Gravity）：设定向心重力系数 `0.015`，防止边缘节点飘失，使其温和回归画布中心。
2. **中枢与子节点连线长度可变（Stretchy Links）**：
   - 摒弃了刚性死板的固定连线长度。
   - **中枢 - 中枢连线**：静止长度设定为较长的 `160px`，弹簧刚度系数 `k = 0.06`，保证三角形核心骨架坚固、空间开阔。
   - **中枢 - 叶子连线**：静止长度设定为短而软的 `95px`，弹簧刚度系数降低为 `0.025`。允许子节点受外部晃动力产生像水母般的弹性呼吸效果。
3. **线不交叉几何算法（Sector Alignment）**：
   - 为避免子节点连线跨越或穿透主三角形的核心连线，我们设定了**外向扇区对齐规则**。
   - 计算三角形重心到各中枢的向量方向（中枢 1: 150°，中枢 2: 30°，中枢 3: -90°，皆朝向三角形外侧）。
   - 折叠展开或初始化时，为每个子节点绑定 `idealAngle` 理想偏向角。
   - 在物理更新循环中增加 `0.03` 的弱引力，拉拢节点回归其所属的非交叉外向扇形区域，从几何上彻底消除主干线交叉。
4. **叶子节点硬碰撞规避（Collision Resolver）**：
   - 在 $O(N^2)$ 排斥循环中引入**硬体碰撞盒逻辑**。依据节点类型半径（中枢 22px，叶子 15px，附加 12px 的安全 Margin），当实际间距 `dist < minDist` 时，进行坐标强制位移排开（纠偏），并扣减碰撞动能阻尼（`0.15`），消除任何节点重叠。
5. **有机 Lissajous（利萨茹）飘移**：
   - 解决节点做单纯圆周轨道打转的机械感，将游走加速度重构为多频合成正余弦波：
     $$dx = \sin(t \cdot 0.35 + \text{hash}) + \cos(t \cdot 0.12 + \text{hash} \cdot 1.5)$$
     $$dy = \cos(t \cdot 0.28 - \text{hash} \cdot 0.7) + \sin(t \cdot 0.18 + \text{hash} \cdot 2.1)$$
   - 设置极低速度 `0.12`，使节点晃动自然灵动。
6. **悬停与拖拽完全静止（Freeze-on-Hover）**：
   - 只要节点处于 `isHovered` 或 `isDragged` 状态，物理循环中将其速度 `vx, vy` 直接清零，并在位置计算中 `continue` 绕过，实现触碰即“钉死”的锚定体验。
   - 在 SVG 的 `<g>` 节点中追加一个隐形大圆（`r=32`）作为热区，防止鼠标轻微抖动导致闪烁滑出。

### 折叠/展开子节点与默认拓扑（中等重要性）

1. **初始状态**：初次载入时，默认将 3 大中枢 ID 添加进展开列表，并在 `initGraph()` 中预先创建 12 个顺序命名的叶子节点（`节点 1` 到 `节点 12`）。
2. **交互行为**：点击中枢节点可以动态从 `nodes` 与 `links` 数组中移除其子节点（折叠），再次点击则根据当前中枢的坐标径向再次泵出（展开）。

---

### Markdown 笔记阅读与编辑器（中等重要性）

1. **右侧滑动抽屉**：抽屉背景适配深浅主题，滑入滑出使用 CSS 贝塞尔曲线过渡。
2. **自研正则 Markdown 解析器**：
   - 编写 `parseMarkdown` 函数，通过正则表达式支持：分级标题、行内/多行代码块、引用、加粗、斜体、列表、以及带斑马纹的 Table 数据报表。
   - 支持数学公式渲染（行内 `\((.+?)\)`，公式块 `$$...$$`）。
   - 解析 GitHub 警示框样式（如 `> [!TIP]`、`> [!IMPORTANT]`），转化为不同边框和渐变背景的彩色警告框。
3. **编辑保存联动**：允许切换为在线编辑文本域，保存后同步更新内存数据库与拓扑图上的 node label。

---

### 本地 Markdown 拖拽解析建档（中等重要性）

1. **拖入捕获**：监听 SVG 容器的拖放事件，拖入时展现高对比度虚线框遮罩。
2. **Frontmatter 解析**：读取 `.md` 文件文本，通过正则剥离头部的 YAML 配置（提取 `title` 和 `tags`）。
3. **自适应网络膨胀**：如果文件包含新标签，自动在物理网络中创建新 Tag 节点和连接线，并展开显示新笔记节点，使其在力导向场中自然弹开并定位。

---

### 视图工具与主题整合（较低重要性 / 仅提及）

1. **工具栏**：支持搜索框对节点名称和标签进行模糊匹配过滤，匹配的节点将在 SVG 中显示动态外发光呼吸脉冲。
2. **状态控制**：支持双击空白处或点击“重置”按钮重新初始化整网物理场，可滚轮无极缩放和拖拽画布平移。
3. **主题切换**：网格线透明度、节点填充及边框色在 Light/Dark 模式间无缝切换。

## 预期结果

1. 页面加载后，以 `0.12` 的柔和速度展现 12 个叶子节点对称且绝不交叉的主干树状网。
2. 鼠标悬停任意节点时，该节点立即瞬间冻结，周围节点受排斥而移开。
3. 点击节点可划出阅读抽屉，富文本样式与主题一致。

## 边界

1. 本阶段为纯前端静态 Mock 演示，所有新建、编辑和删除的笔记修改皆保存在前端内存变量中，刷新页面即会重置，暂不与后端 API/MySQL 建立持久化同步。
